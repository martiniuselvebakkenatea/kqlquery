# SecurityEvent 24h Volume Usage

## Description

This Kusto Query Language (KQL) script queries the `SecurityEvent` table in Microsoft Sentinel to identify **the total number of events and the total data volume generated by SecurityEvent logs during the last 24 hours**.
It filters for **events generated in the past 24 hours** and summarizes the results to calculate both the **event count** and the **total billed data size**. The billed size is converted from bytes to **megabytes (MB)** for easier readability.

## Purpose

* **Data volume monitoring:** Measure how much `SecurityEvent` data is being ingested in the last 24 hours.
* **Cost visibility:** Understand ingestion costs since `_BilledSize` represents billable data volume.
* **Connector monitoring:** Verify that Windows Security Event logs are actively flowing into Sentinel.
* **Capacity planning:** Estimate daily log ingestion growth and plan storage or retention accordingly.

## Data Source

* **Log Table:** `SecurityEvent`
* **Key Fields Used:** `TimeGenerated`, `_BilledSize`, `EventID`

## KQL Query

```kusto
SecurityEvent
| where TimeGenerated > ago(24h)                 // Only include events from the last 24 hours
| summarize TotalEvents = count(), TotalSizeBytes = sum(_BilledSize)
| extend TotalSizeMB = round(TotalSizeBytes / 1024 / 1024, 2)  // Convert bytes to MB
| project TotalEvents, TotalSizeMB
```

### Query Explanation

| Step                             | Explanation                                     |
| -------------------------------- | ----------------------------------------------- |
| `where TimeGenerated > ago(24h)` | Filters logs generated within the last 24 hours |
| `count()`                        | Calculates the total number of events           |
| `sum(_BilledSize)`               | Calculates total billable ingestion size        |
| `extend TotalSizeMB`             | Converts the total size from bytes to MB        |
| `project`                        | Returns only the relevant output fields         |
