# SecurityEvent Event ID Data Volume (Last 24 Hours)

## Description

This Kusto Query Language (KQL) script queries the `SecurityEvent` table in Microsoft Sentinel to calculate the **total ingestion size generated by a specific Windows Event ID during the last 24 hours**.
It filters for a **specific `EventID`** within the last **24-hour time window** and summarizes the results by calculating the **total billed data size**.

The query converts the total ingestion size from bytes into **megabytes (MB)** for easier interpretation.

## Purpose

* **Data volume analysis:** Determine how much Sentinel ingestion volume is generated by a specific Windows event type.
* **Cost monitoring:** Identify events that contribute significantly to Log Analytics ingestion costs.
* **Connector tuning:** Help determine whether certain Windows events should be filtered or optimized.
* **SIEM optimization:** Support decisions about data retention, filtering, or transformation rules.

## Data Source

* **Log Table:** `SecurityEvent`
* **Key Fields Used:** `TimeGenerated`, `EventID`, `_BilledSize`

## KQL Query

```kusto
SecurityEvent
| where TimeGenerated > ago(24h)           // Limit results to the last 24 hours
| where EventID == 0000                    // Replace with the Windows Event ID you want to analyze
| summarize TotalSizeBytes = sum(_BilledSize)
| extend TotalSizeMB = round(TotalSizeBytes / 1024 / 1024, 2)
| project TotalSizeMB
```

### Query Explanation

| Step                       | Explanation                                 |
| -------------------------- | ------------------------------------------- |
| `TimeGenerated > ago(24h)` | Filters logs generated in the last 24 hours |
| `EventID == 0000`          | Targets a specific Windows event ID         |
| `sum(_BilledSize)`         | Calculates total ingestion size in bytes    |
| `extend TotalSizeMB`       | Converts bytes into megabytes               |
| `project TotalSizeMB`      | Outputs the final calculated data volume    |
